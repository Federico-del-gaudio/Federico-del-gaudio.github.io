PUMMEL

Solo project

Objective: Combat design, Enemy AI, animation & blueprints

About the Project

This is one of my most complete and in-depth projects developed in Unreal Engine 5. For this project, I aimed to explore and experiment with PaperZD, a 2D animation plugin for UE5. My main focus was on integrating flipbook animations, which I sourced online, and using them to bring characters to life in a fluid and responsive way.

I implemented a combo system that reacts to player input timing, allowing for a more dynamic combat experience. I also worked extensively with the Animation Controller, setting up states and transitions to ensure smooth and responsive character behavior based on movement and attack actions.

Additionally, I developed core gameplay systems such as damage handling and health management, laying the foundation for combat interactions and player-enemy feedback. This project gave me the opportunity to deepen my knowledge of UE5's 2D tools while also strengthening my ability to structure gameplay logic and animation flow in a cohesive and scalable way.

Implementation Insights - PaperZD & 2D Animation

Key Learnings from PaperZD Implementation

Working with PaperZD for 2D animation in Unreal Engine 5 taught me valuable lessons about animation workflow and performance optimization. The plugin bridges the gap between traditional 2D animation and modern game engines.

The most significant insight was understanding how flipbook animations work within a 3D engine context. I learned to optimize sprite sheets and manage animation states efficiently to maintain smooth performance.

Another crucial learning was the integration of 2D assets into a 3D pipeline. This required careful planning of coordinate systems, depth sorting, and camera management to create a cohesive visual experience.

Finally, I discovered the importance of modular animation systems. Creating reusable animation blueprints and state machines allowed for faster iteration and more consistent character behavior.

Technical Challenge

Implementing responsive combo chains required precise input buffering and animation canceling systems to ensure fluid gameplay while maintaining visual feedback.

Combo System Design Insights

Developing the combo system for Pummel taught me valuable lessons about player input timing and animation blending. The system needed to feel responsive while maintaining visual coherence.

One key insight was the importance of input buffering. Creating a window where player inputs could be queued for execution made the combat feel more forgiving and accessible without sacrificing depth.

I learned that animation canceling needed careful balancing. Allowing too much canceling made animations feel disconnected, while too little made the controls feel sluggish.

The integration of hit stop effects and screen shake was particularly important for creating satisfying impact feedback. Small visual and auditory cues dramatically improved the perceived weight of attacks.
